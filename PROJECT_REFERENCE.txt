ARCHITECTURE STATISTIQUES (v1.3)
---------------------------------
**DÉCISION IMPORTANTE** : Architecture décentralisée

Principe :
- Chaque écran gère ses propres statistiques
- Utilisation de SharedPreferences pour la persistance
- Pas de state management global (Provider, Riverpod, etc.)
- Autonomie et maintenabilité de chaque fonctionnalité

Implémentation :
1. Scénarios Commerciaux (commercial_scenario_screen.dart) :
   - Gère ses propres stats Chifoumi et réussites
   - Clés SharedPreferences : 'chifoumi_*' et 'scenario_*'

2. Pannes - Mode Créateur (malfunction_creator_screen.dart) :
   - Gère ses propres stats de pannes tirées
   - Clés SharedPreferences :
     * 'creator_total_drawn'
     * 'creator_easy_drawn'
     * 'creator_medium_drawn'
     * 'creator_hard_drawn'

3. Pannes - Mode Dépanneur (malfunction_home_screen.dart) :
   - Conserve les stats du mode dépanneur
   - Clés SharedPreferences :
     * 'malfunction_tech_easy_success'
     * 'malfunction_tech_easy_total'
     * 'malfunction_tech_medium_success'
     * 'malfunction_tech_medium_total'
     * 'malfunction_tech_hard_success'
     * 'malfunction_tech_hard_total'

Avantages :
- Pas de prop drilling
- Maintenance simplifiée
- Chaque fonctionnalité est autonome
- Peut évoluer vers state management global si nécessaire

STRUCTURE DES SCÉNARIOS
------------------------
Chaque scénario contient :
- id : numéro unique (1-100)
- clientProfile : type de client
- clientRequest : demande formulée
- budgetInfo : contrainte budgétaire
- clientAttitude : comportement/état d'esprit
- difficulty : easy/medium/hard
- keyQuestions : questions à poser au client
- solutions : produits/services à proposer (avec prix, avantages, inconvénients, URL)
- commonTraps : pièges à éviter
- skillsWorked : compétences RNCP travaillées (IDI/ADRN/TIP)

STRUCTURE DES PANNES
---------------------
Chaque panne contient :
- id : numéro unique
- name : nom de la panne
- description : description détaillée
- symptoms : symptômes observables (liste)
- category : hardware/software/setup/network/printer/peripheral
- difficulty : easy/medium/hard
- estimatedTime : temps estimé de création
- creationSteps : procédure de création (liste)
- creationTips : conseils de simulation (liste)
- skillsWorked : compétences RNCP mobilisées (IDI/ADRN/TIP)
